Autômatos: criando uma simulação de robô em Marte

Na unidade "Autômatos: criando uma simulação de um Robô em Marte", os alunos constroem um jogo interativo usando programação em blocos, no qual controlam um robô explorador que percorre o planeta Marte em busca de pontos de pesquisa. O projeto é desenvolvido passo a passo, com ênfase no raciocínio lógico, na resolução de problemas e no uso criativo de comandos básicos de programação visual.

Durante a construção do jogo, os estudantes aprendem a criar um cenário espacial e programar o movimento do robô usando as teclas W, A e D. Variáveis são utilizadas para armazenar informações como a energia do robô e os pontos de pesquisa coletados, promovendo a noção de estado e pontuação dentro do jogo. Também são aplicadas estruturas condicionais (como se/senão) para controlar os limites da tela e para lidar com interações entre o robô e outros elementos, como tempestades e fontes de energia.

A lógica do jogo se expande com o uso de clones — cópias programadas de objetos como nuvens ou insetos — que representam obstáculos ou elementos interativos. Os alunos aprendem a criar e gerenciar esses clones, otimizando o código e tornando a simulação mais dinâmica e desafiadora.

Além disso, são introduzidos efeitos sonoros, telas de instruções e uma interface de início, tornando o projeto mais completo. Ao longo das aulas, o conceito de autômatos é apresentado, conectando a programação do jogo com o funcionamento de máquinas automáticas do cotidiano. Por fim, o curso aborda noções de cidadania digital, alertando para o uso consciente de imagens e a proteção de dados pessoais.

Ao concluir a unidade, os alunos terão desenvolvido um jogo funcional e original, aplicando de forma prática os fundamentos da programação em blocos e compreendendo como autômatos podem ser simulados por meio de lógica computacional.

Lógica de programação: criando arte interativa com P5.js

Na unidade "Arte com código: crie projetos visuais com P5.js", os alunos exploram de forma criativa a interseção entre arte e programação. Utilizando a biblioteca P5.js, são introduzidos à criação de projetos visuais e interativos, que unem formas geométricas, cores e lógica de programação para gerar composições dinâmicas na tela.

O curso começa com os primeiros passos na plataforma, onde os estudantes criam suas contas e escrevem suas primeiras linhas de código. A função createCanvas() é utilizada para definir o espaço de desenho, enquanto os comandos setup() e draw() estruturam a lógica do programa. A partir disso, o projeto evolui com a criação de formas geométricas (retângulos, elipses, linhas), posicionadas usando o plano cartesiano.

Os estudantes também aprendem a aplicar cores aos elementos gráficos com comandos específicos da P5.js, descobrindo como pintar formas e linhas e como controlar a aparência visual da tela. A lógica condicional, com o uso do comando if, começa a ser integrada ao projeto, permitindo a criação de interações básicas entre os elementos do desenho.

Um dos momentos mais marcantes do curso é a aplicação da função map(), que permite transformar valores de uma escala para outra — técnica usada, por exemplo, para simular o movimento de olhos que seguem o cursor, criando um "efeito Monalisa". Esse recurso destaca como conceitos matemáticos, como o teorema de Pitágoras e mapeamento de valores, podem ser aplicados para criar arte digital interativa.

Ao final da unidade, os estudantes não apenas dominam os fundamentos da biblioteca P5.js, como também desenvolvem uma nova percepção de como a matemática, a lógica e a arte podem se encontrar no universo do código.

Listas, funções e repetição: criando um jogo de corrida

Na unidade "Jogo de Corrida com P5.js", os alunos aprendem a criar um jogo interativo de corrida, utilizando a biblioteca de programação visual P5.js. Ao longo do curso, desenvolvem habilidades fundamentais de programação, como uso de variáveis, laços de repetição, estruturas condicionais e manipulação de eventos do teclado e do mouse.

O projeto começa com a criação da pista e dos personagens corredores, usando comandos gráficos básicos. Logo no início, os estudantes implementam a lógica para movimentar os corredores utilizando teclas específicas do teclado, introduzindo a função keyReleased() para capturar eventos de interação e tornar os movimentos mais fluidos e controláveis.

A lógica do jogo é aprofundada com o uso de estruturas condicionais como if, para detectar qual jogador cruzou a linha de chegada primeiro e declarar o vencedor da partida. Para isso, os alunos precisam relacionar coordenadas no canvas com as ações dos jogadores

À medida que o jogo avança, o código é refatorado com a criação de funções personalizadas. Essa prática de modularização facilita a leitura e organização do código, preparando os estudantes para projetos mais complexos. Também são introduzidas listas (arrays), permitindo a adição de múltiplos jogadores e a otimização do código com laços de repetição, como o for.

Os alunos são incentivados a personalizar o jogo com diferentes personagens, explorando o uso de emojis, textos e cores para representar os corredores. A cada nova etapa, o projeto se torna mais completo, com múltiplos competidores, animações e reconhecimento automático de padrões para melhorar o desempenho visual e lógico da aplicação.

O projeto final inclui uma corrida com até quatro jogadores, uma linha de chegada construída com repetições visuais, e um sistema para detectar o vencedor de forma automatizada. O jogo também pode ser compartilhado com colegas, promovendo a troca de experiências e o orgulho pela criação.

Ao concluir essa unidade, os estudantes desenvolvem uma compreensão sólida da lógica de programação, aprendem a aplicar conceitos matemáticos no contexto visual e descobrem como transformar código em uma experiência divertida e interativa.
